# 당골래 (Dangolrae) PRD

## MVP 캡슐

```
1. 목표: 전국 기도터 정보를 지도 기반으로 탐색할 수 있는 플랫폼 구축
2. 페르소나: 무속 신앙 관심자, 기도터 방문자, 무속인
3. 핵심 기능: FEAT-1 기도터 디렉토리, FEAT-2 지도 탐색, FEAT-3 큐레이션 콘텐츠
4. 성공 지표 (노스스타): 월간 기도터 상세 페이지 조회수
5. 입력 지표: 등록 기도터 수, 검색 전환율
6. 비기능 요구: 지도 로딩 2초 이내, 모바일 퍼스트
7. Out-of-scope: 후기 시스템, 즐겨찾기, 회원가입/로그인(사용자), 줄/신격 태깅
8. Top 리스크: 초기 데이터 부족 (300건 미만 시 지도 빈 영역 다수)
9. 완화/실험: 공공데이터(전통사찰) + 수작업 입력으로 300건 이상 확보 후 런칭
10. 다음 단계: 사용자 인증 + 후기 시스템 (v2)
```

---

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 당골래 (Dangolrae) |
| 서비스 정의 | 전국의 기도터·굿당·사찰·서낭당·산신당을 지도 기반으로 탐색하는 플랫폼 |
| 핵심 가치 | 흩어진 기도터 정보를 정형화하여 신뢰성 있는 탐색 경험 제공 |
| 타겟 사용자 | 무속 신앙 관심자, 기도터 방문자, 무속인, 민속신앙 커뮤니티 이용자 |

## 2. 프로젝트 배경

한국 민속 신앙 및 기도 문화는 풍부하지만, 기도터 정보에는 다음 구조적 문제가 존재한다:

| 문제 | 현황 |
|------|------|
| 정보 산재 | 블로그·카페에 흩어져 있어 탐색이 어려움 |
| 위치 정보 부정확 | 지번 주소만 존재하거나 지도 연동 없음 |
| 방문 정보 부족 | 예절, 준비물, 분위기 등 실용 정보 부재 |
| 경험 공유 미흡 | 방문자 후기 플랫폼 부재 |

## 3. 핵심 기능 (MVP)

### FEAT-1: 기도터 디렉토리
- **설명**: 전국 기도터 정보를 정형 데이터베이스로 제공
- **Why**: 기도터 정보가 블로그/카페에 산재되어 탐색이 어렵기 때문
- **우선순위**: MVP 필수
- **데이터 필드**: 이름, 주소, 위도/경도, 유형, 설명, 이미지, 연락처
- **유형 분류**: 사찰 | 굿당 | 서낭당 | 산신당 | 당산목 | 기타
- **참고**: 줄(계열)/신격 태깅은 v2로 미룸 (초기엔 단순 유형 분류만)

### FEAT-2: 지도 기반 탐색
- **설명**: 카카오맵 기반 마커 탐색 + 필터링 + 내 주변 탐색
- **Why**: 위치 기반 탐색이 기도터 찾기의 핵심 경험
- **우선순위**: MVP 필수
- **세부 기능**:
  - 전체 마커 표시 (유형별 커스텀 아이콘)
  - 지도 영역 기반 실시간 갱신
  - 마커 클릭 → 바톰시트 미리보기 → 상세 이동
  - 클러스터링 (마커 밀집 시)
  - 유형 필터 / 지역 필터 / 정렬
  - GPS 기반 내 주변 탐색 (1km/3km/5km)

### FEAT-3: 큐레이션 콘텐츠
- **설명**: 관리자가 편성하는 기획 그룹 리스트
- **Why**: 빈약한 초기 데이터를 보완하고 탐색 경험 강화
- **우선순위**: MVP 필수
- **큐레이션 유형**: 편집 추천, 명산 기도터, 지역별 추천, 신규 등록

## 4. 사용자 스토리

| As a | I want | So that |
|------|--------|---------|
| 기도터 방문자 | 내 근처 기도터를 지도에서 찾고 싶다 | 가까운 기도터를 빠르게 발견할 수 있다 |
| 무속 신앙 관심자 | 기도터 유형별로 필터링하고 싶다 | 원하는 유형의 기도터만 볼 수 있다 |
| 방문 예정자 | 기도터의 상세 정보(주소, 사진)를 보고 싶다 | 방문 전 미리 정보를 파악할 수 있다 |
| 관리자 | 기도터 정보를 등록/수정/삭제하고 싶다 | 정확한 데이터를 유지할 수 있다 |
| 관리자 | 큐레이션 리스트를 편성하고 싶다 | 사용자에게 가치 있는 탐색 경험을 제공할 수 있다 |

## 5. 경쟁사 분석

| 구분 | 기존 서비스(블로그/카페) | 경쟁사(아리톡 등) | 당골래 |
|------|------------------------|-------------------|--------|
| 위치 정보 | 텍스트 주소만 | 주소 텍스트 기반 | 위도/경도 좌표 + 지도 마커 |
| 분류 체계 | 없음 | 지역별 단순 분류 | 유형 분류 (MVP) + 줄/신격 태깅 (v2) |
| 지도 탐색 | 불가 | 제한적 | 카카오맵 완전 연동 |
| DB 구조 | 없음 | 게시판(그누보드) | 정형 DB 스키마(Supabase) |

## 6. 가설 vs 사실

| 항목 | 가설/사실 | 검증 방법 |
|------|-----------|-----------|
| 기도터 정보 니즈 존재 | 가설 | MVP 출시 후 조회수/체류시간 측정 |
| 300건이면 전국 커버리지 가능 | 가설 | 시/도별 분포 확인 후 빈 지역 파악 |
| 공공데이터로 100건 확보 가능 | 사실 | data.go.kr 전통사찰 인허가 데이터 확인 완료 |
| 카카오맵 연동이 핵심 | 가설 | 사용자 행동 데이터로 지도 vs 목록 이용률 비교 |

## 7. MVP 범위

### MVP 포함
- 기도터 디렉토리 (조회/상세)
- 카카오맵 탐색 + 마커 + 클러스터링
- 유형 필터링 + 지역 필터링
- 내 주변 기도터 탐색 (GPS)
- 큐레이션 섹션
- 관리자 페이지 (기도터/큐레이션/배너 관리)

### MVP 제외 (v2)
- 사용자 회원가입/로그인
- 후기 시스템 (작성/조회/공감)
- 즐겨찾기
- 마이페이지
- 줄(계열)/신격 태깅
- 기도터 사용자 제보
- 방문 기록 캘린더
- 인기 기도터 랭킹
